네트워크 구축할 때 인터페이스끼리 연결해서 아이피 할당하고 라우팅 설정하는 부분의 쉘 스크립트다. 범위는 물리, 데이터링크, 네트워크 계층까지다. 이런 식으로 네트워크를 구축한 뒤에는 바로 애플리케이션 프로그래밍할 수 있다.

ble_host.sh
라우터에 접속하는 slave 측이 이것을 실행한다. 실행 효과는 두 가지다.
1. 블루투스 장치에 로우팬을 활성화시키기 위한 커널 모듈을 적재한다.
2. 적재한 커널 모듈을 활성화한다.
이제 ble로 라우터에 연결하면 ble 인터페이스임에도 아이피 주소가 할당된다! 다시 말해 로우팬 덕택에 IP 주소를 할당 받아서, ble로 인터넷을 쓸 수 있게 되었다.

1_ble_router.sh
라우터가 되는 master 측이 이것을 실행한다. 실행 효과를 번호 매겨 적으면 다음과 같다.
1. RA 데몬을 실행시켜 미리 설정해 둔 IPv6 prefix를 네트워크에 뿌린다. 이렇게 하면 네트워크에 연결된 장비들이 IPv6 주소를 할당 받게 된다. 이런 IPv6 할당 방법을 "IPv6 stateless auto-configuration"이라고 부른다.
2. 블루투스 장치에 로우팬을 활성화시키기 위한 커널 모듈을 적재한다.
3. 적재한 커널 모듈을 활성화한다.
4. 이 라우터의 ble를 다른 장비에서 discover 가능하게 만든다. (le adv)

2_ble_router.sh
직전 것(1_ble_router.sh)을 실행한 뒤에 slave가 하나라도 접속한 상태에서 실행한다. 블루투스 연결이 있어야 bt0 인터페이스가 활성하기 때문이다. 실행 효과는 두 가지다.
1. 이 라우터의 ble를 다른 장비에서 discover 가능하게 만든다. (le adv)
2. 리눅스 커널이 IPv6 포워딩 활성화 중이기 때문에 라우터 자기 자신의 인터페이스들은 RA의 prefix를 부여 받지 않는다. 그 때문에 수동으로 글로벌 IPv6 주소를 할당해 준다.

conf_6lbr.sh
6lbr를 겨우 설치한 뒤 한 번만 필요한 설정을 수행한다.

run_6lbr.sh
6lbr과 radvd가 바로 사용할 수 있도록 온전히 설치된 후에 이것을 실행해서 라우팅과 브리징을 수행한다. (1_ble_router.sh)의 수행 뒤에 실행할 것이 권장된다.
